Structure of a Pixi Project

  • renderer manages <canvas> element
  • scene = PIXI.Container object
  • add visual elements via .addChild
  • render!

Animations

  • change scene
  • render
  • request new animation frame
  • repeat!

Pixi Elements

  • Container: contains other elements
  • Graphics: vector graphics
  • Sprite: image with texture
  • Text: text element, internally rendered as a Sprite

Scene-Graph

  • all elements are also Containers!
  • children are positioned in parent coordinates
  • transformations are applied to all children

External Resources

  • loaded asynchronously via PIXI.loaders.Loader
  • for images and sprite sheets
  • loaded images are automatically place in PIXI.utils.TextureCache
  • Sprites can be created via PIXI.Sprite.fromFrame

Interactions

  • elements must have property .interactive=true
  • node-style events (.on, .once...)
  • event names corresppond to browser events (mousemove, click...)
  • touch events are supported!

Tests

  • Pixi needs a DOM featuring a real <canvas> element!
  • Karma + Mocha work well, even in PhantomJS
